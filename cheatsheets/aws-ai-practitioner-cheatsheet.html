ML Concepts, Algorithms & Architectures

💡
GPT
Generative Pre-trained Transformer
TRANSFORMER
Decoder-only transformer architecture pre-trained on massive text corpora using next-token prediction (causal language modeling). Foundation of modern LLMs like ChatGPT, Claude, and Amazon Titan Text.
TypeAutoregressive decoder-only transformer
TrainingUnsupervised pre-training on next-token prediction → RLHF fine-tuning for alignment
Key IdeaSelf-attention allows each token to attend to all previous tokens; scales with data and parameters
Use Cases
Text generationSummarizationCode genQ&ATranslationChatbots
AWS LinkAmazon Titan Text (Bedrock), Claude via Bedrock, Amazon Q Developer are GPT-style models
VersionsGPT-1 (2018) → GPT-2 → GPT-3 (175B params) → GPT-3.5 → GPT-4 (multimodal)
⚡ EXAM TIPGPT = decoder-only, generative. Key mechanism: causal self-attention (each token only sees past). Contrast with BERT (encoder-only, bidirectional). Prompt engineering works because GPT continues your text.
🔬
BERT
Bidirectional Encoder Representations from Transformers
TRANSFORMER
Encoder-only transformer by Google that reads text bidirectionally (left AND right context simultaneously), pre-trained via masked language modeling (MLM) and next sentence prediction (NSP).
TypeBidirectional encoder-only transformer
Pre-trainingMasked Language Model (MLM) — predict masked tokens; Next Sentence Prediction (NSP)
Fine-tuningAdd task-specific head on top of [CLS] token representation
Use Cases
Sentiment analysisNERQuestion answeringText classificationSemantic search
VariantsRoBERTa (improved training), DistilBERT (smaller/faster), ALBERT, BioBERT (medical)
AWS LinkBERT models available in SageMaker JumpStart; used in Amazon Kendra internally
⚡ EXAM TIPBERT = understanding tasks (classification, NER, QA). GPT = generation tasks. BERT reads bidirectionally so it understands context better. Not suitable for text generation. SageMaker JumpStart hosts BERT variants.
⚙️
Transformer
Attention Is All You Need (Vaswani et al., 2017)
ARCHITECTURE
Revolutionary architecture that replaced RNNs for sequence tasks. Uses self-attention to model relationships between all tokens simultaneously — enabling massive parallelism and long-range dependencies.
Core MechanismMulti-head self-attention + feed-forward layers + positional encoding + layer norm
Attention FormulaAttention(Q,K,V) = softmax(QKᵀ / √dₖ) · V — queries, keys, values from input
Advantage over RNNParallel processing (vs sequential), no vanishing gradient, O(1) path length between any two tokens
Variants
Encoder-only (BERT)Decoder-only (GPT)Encoder-Decoder (T5, BART)
Use CasesNLP (translation, summarization, QA), vision (ViT), audio, code, multimodal
⚡ EXAM TIPTransformer = the backbone of ALL modern LLMs. Self-attention = each token looks at all others. Positional encoding = adds word order info (transformers have no inherent sequence order). Know encoder vs decoder variants.
🔁
RNN / LSTM / GRU
Recurrent Neural Networks & variants
ARCHITECTURE
Sequential neural networks that maintain a hidden state across time steps, making them naturally suited for time-series and sequential data. LSTMs and GRUs solve the vanishing gradient problem of vanilla RNNs.
RNNBasic recurrent unit; suffers from vanishing/exploding gradients for long sequences
LSTMLong Short-Term Memory — gates (input, forget, output) control what to remember/forget over long sequences
GRUGated Recurrent Unit — simplified LSTM with fewer parameters; often comparable performance
Use Cases
Time-series forecastingSpeech recognitionText generation (pre-transformer)Anomaly detectionMusic generation
LimitationSequential processing = slow training; largely superseded by Transformers for NLP tasks
AWS LinkDeepAR+ in Amazon Forecast uses LSTM; SageMaker supports RNN training
⚡ EXAM TIPRNNs process sequences step-by-step (slow). LSTMs fix vanishing gradients with memory gates. Transformers replaced RNNs for most NLP. But RNNs/LSTMs still appear in time-series (DeepAR in Forecast uses them).
🖼️
CNN / ResNet
Convolutional Neural Networks & Residual Networks
ARCHITECTURE
CNNs use convolutional filters to extract spatial hierarchies from images (edges → textures → objects). ResNet introduced skip connections (residual connections) allowing training of very deep networks (100+ layers).
CNN LayersConv → ReLU → Pooling → Flatten → FC layers. Filters learn spatial features automatically.
ResNetResidual connections: output = F(x) + x. Skip connections prevent vanishing gradients in deep networks.
Popular CNNs
ResNet-50/101VGG-16EfficientNetMobileNetInception
Use Cases
Image classificationObject detectionSegmentationMedical imagingOCR
Transfer LearningPre-trained ResNet on ImageNet → fine-tune last layers on your dataset (requires far less data)
AWS LinkSageMaker Image Classification uses ResNet; Rekognition uses CNN architectures internally
⚡ EXAM TIPCNNs = computer vision. ResNet solved "degradation problem" with skip connections. Transfer learning with pre-trained CNNs = train custom vision models with small datasets. SageMaker Image Classification algorithm = built-in ResNet.
🎭
GAN
Generative Adversarial Network
GENERATIVE
Two neural networks compete in a minimax game: a Generator creates fake data to fool the Discriminator; the Discriminator learns to distinguish real from fake. Through adversarial training both networks improve until the generator produces realistic outputs.
GeneratorTakes random noise (latent vector z) as input → outputs synthetic data (images, audio, etc.)
DiscriminatorBinary classifier: real vs fake. Provides training signal to the Generator.
Variants
DCGANStyleGANCycleGANPix2PixWGANBigGAN
Use Cases
Image synthesisData augmentationStyle transferDeepfakesDrug discoverySuper-resolution
ChallengesMode collapse, training instability, difficult hyperparameter tuning
AWS LinkStable Diffusion on Bedrock uses diffusion (related generative approach); SageMaker trains GANs
⚡ EXAM TIPGANs = adversarial generative models. Generator vs Discriminator = zero-sum game. Mode collapse = generator produces limited variety. Largely superseded by diffusion models for images. GAN training is notoriously unstable.
🌌
Diffusion Models
Denoising Diffusion Probabilistic Models (DDPM)
GENERATIVE
Generate data by learning to reverse a gradual noising process. Forward process adds Gaussian noise until data becomes pure noise; reverse process (denoising) learns to reconstruct the original. Powers Stable Diffusion and DALL-E 2.
ForwardGradually add noise over T steps: x₀ → x₁ → ... → xₜ ~ N(0,I)
ReverseNeural network (U-Net) learns to predict and remove noise: xₜ → xₜ₋₁ → ... → x₀
ConditioningText-conditioned via CLIP embeddings enables text-to-image generation
Use Cases
Image generationImage editingInpaintingVideo synthesisDrug molecule design
AWS LinkStable Diffusion available on Amazon Bedrock (Stability AI) and SageMaker JumpStart
⚡ EXAM TIPDiffusion models = current SOTA for image generation. More stable training than GANs. Stable Diffusion on Bedrock uses this architecture. Latent Diffusion Models (LDMs) operate in compressed latent space for efficiency.
🗜️
VAE
Variational Autoencoder
GENERATIVE
Generative model that learns a compressed probabilistic latent representation of data. Encoder maps input to a latent distribution; Decoder samples from it to reconstruct — enabling both compression and generation.
EncoderMaps input x to latent distribution parameters: μ (mean) and σ (std) → z ~ N(μ,σ²)
DecoderReconstructs data from sampled z; trained with reconstruction loss + KL divergence regularization
vs GANVAE = stable training, blurry outputs; GAN = unstable training, sharp outputs
Use Cases
Anomaly detectionImage generationData compressionDrug discoveryLatent space interpolation
AWS LinkVAEs used internally in Stable Diffusion's latent space; trainable on SageMaker
⚡ EXAM TIPVAE = probabilistic autoencoder. Latent space is continuous and smooth — can interpolate between data points. Great for anomaly detection (high reconstruction error = anomaly). KL divergence term regularizes latent space.
✂️
SVM
Support Vector Machine
CLASSICAL ML
Supervised learning algorithm that finds the optimal hyperplane maximizing the margin between classes. Uses the kernel trick to handle non-linearly separable data by mapping to higher-dimensional spaces.
Core IdeaMaximize margin between classes. Support vectors = data points closest to the decision boundary.
Kernel TrickRBF, polynomial, sigmoid kernels transform data to higher dimensions for non-linear classification
StrengthsEffective in high dimensions, works well with small datasets, robust to outliers
WeaknessesSlow on large datasets, kernel/C/gamma hyperparameter sensitivity, no probability output by default
Use Cases
Text classificationImage classificationBioinformaticsAnomaly detection (One-Class SVM)
AWS LinkAvailable in SageMaker built-in algorithms; scikit-learn SVM trainable on SageMaker
⚡ EXAM TIPSVM = classical ML for classification/regression. C parameter controls bias-variance tradeoff (larger C = less regularization). One-Class SVM = novelty/anomaly detection. Mostly replaced by tree ensembles and DL for tabular data.
🏆
XGBoost
Extreme Gradient Boosting
CLASSICAL ML
Highly optimized gradient boosting library that sequentially builds an ensemble of decision trees, each correcting errors of the previous. Dominant algorithm for structured/tabular data in ML competitions and industry.
How It WorksAdditive training: each new tree fits the residual errors (negative gradients) of the ensemble so far
Key Params
n_estimatorsmax_depthlearning_rate (eta)subsamplecolsample_bytreelambda/alpha (L2/L1)
StrengthsHandles missing values natively, built-in regularization, fast (parallel tree construction), feature importance
Use Cases

Random Forest
Ensemble of Decision Trees (Bagging)
CLASSICAL ML
Builds many decision trees in parallel on random data subsets (bagging) and random feature subsets, then aggregates predictions via majority vote (classification) or averaging (regression).
BaggingBootstrap Aggregating — each tree trained on random sample with replacement (reduces variance)
Feature SamplingEach split considers only √n features (classification) or n/3 (regression) — decorrelates trees
vs XGBoostRF = parallel (bagging); XGBoost = sequential (boosting). RF more robust to overfitting by default.
Use Cases

AWS LinkSageMaker scikit-learn container; used internally by SageMaker Autopilot
⚡ EXAM TIPRandom Forest = bagging ensemble (parallel). XGBoost = boosting ensemble (sequential). RF reduces variance; Boosting reduces bias. RF less prone to overfitting but generally outperformed by XGBoost on tabular data.

WaveNet
Deep Generative Audio Model (DeepMind)
AUDIO / GENERATIVE
Autoregressive generative model for raw audio waveforms using dilated causal convolutions. Generates audio sample-by-sample at 16kHz+, producing human-like speech — the architecture behind Google's text-to-speech.
Key InnovationDilated causal convolutions — exponentially growing receptive field without losing temporal resolution
AutoregressiveEach audio sample conditioned on all previous samples p(x) = ∏ p(xₜ | x₁...xₜ₋₁)
ArchitectureStack of dilated conv layers (dilation: 1,2,4,8,...,512) × multiple blocks + gated activations
Use Cases
Text-to-speechMusic generationVoice cloningAudio synthesis
LimitationSlow inference (generates one sample at a time at 16,000 samples/sec); WaveRNN and parallel variants address this
AWS LinkAmazon Polly uses neural TTS architectures inspired by WaveNet for NTTS voices
⚡ EXAM TIPWaveNet = neural raw audio generation. Dilated convolutions = large receptive field efficiently. Amazon Polly NTTS (Neural TTS) uses similar deep learning for lifelike speech. Autoregressive generation = slow but high quality.

Embeddings convert text/images/data into dense numerical vectors capturing semantic meaning. Similar items cluster together in vector space. RAG combines retrieval (find relevant embeddings) with generation (LLM answers using retrieved context).
Embeddingstext-embedding-3 (OpenAI), Amazon Titan Embeddings (Bedrock) — map text to 1536-dim vectors
Vector DB

Fine-Tuning & RLHF
Model Adaptation & Reinforcement Learning from Human Feedback
TRAINING
Fine-tuning adapts pre-trained models to specific tasks/domains with labeled data. RLHF aligns LLMs to human preferences using a reward model trained on human comparisons, then RL (PPO) to optimize for reward.
Full Fine-tuneUpdate all model weights on domain-specific data. Expensive. Best for significant distribution shift.
LoRA/PEFTLow-Rank Adaptation — freeze base model, train small adapter matrices. 10-100x more efficient.
RLHF Steps1. SFT (supervised fine-tune). 2. Train reward model on human preference data. 3. PPO to maximize reward.
RLAIFRL from AI Feedback — use another LLM as judge instead of human annotators (cheaper, faster)
AWS LinkBedrock Fine-tuning (custom models), SageMaker Training Jobs, SageMaker JumpStart fine-tuning
⚡ EXAM TIPFine-tuning vs RAG: Fine-tuning = bakes knowledge into weights (costly, for style/format/domain). RAG = retrieves knowledge at inference (cheaper, for up-to-date facts). Bedrock supports both fine-tuning and Knowledge Bases (RAG).

Bias–Variance Tradeoff
Overfitting, Underfitting & Regularization
FUNDAMENTALS
Core ML tradeoff: high bias = underfitting (model too simple, misses patterns); high variance = overfitting (model memorizes training data, fails on new data). Goal: find the sweet spot.
UnderfittingHigh training error + high validation error. Fix: more complex model, more features, less regularization.
OverfittingLow training error + high validation error. Fix: more data, dropout, L1/L2 regularization, early stopping, cross-validation.
Regularization
L1 (Lasso) — sparsityL2 (Ridge) — weight decayDropoutEarly stoppingData augmentation
Cross-Validationk-fold CV: split data into k folds, train on k-1, test on 1, rotate — more reliable evaluation than single split
AWS LinkSageMaker Automatic Model Tuning (HPO) searches for optimal regularization hyperparameters
⚡ EXAM TIPMassive exam topic. Train accuracy high, test accuracy low → overfitting. Both low → underfitting. Dropout = randomly zero out neurons during training (regularization). L1 = feature selection. L2 = shrinks weights. SageMaker HPO automates hyperparameter search.

RegressionMAE (mean absolute error), MSE, RMSE, R² (coefficient of determination), MAPE
LLM / NLP

Imbalanced DataAccuracy misleading → use F1, Precision-Recall AUC, or Matthews Correlation Coefficient
AWS LinkBedrock Model Evaluation tracks ROUGE, BERTScore; SageMaker Clarify reports bias metrics
⚡ EXAM TIPPrecision vs Recall tradeoff: High precision = few false alarms. High recall = catch everything (fewer misses). F1 = harmonic mean. For fraud detection, Recall matters more (don't miss fraud). BLEU/ROUGE = NLP generation quality. Bedrock Model Evaluation uses these.
